# BadPunPC 解题策略 — 活文档（v3，29 题蒸馏版）

> **使命**：把每道题的经验沉淀下来，让 solver.py 的 SYSTEM_PROMPT 越来越强。
> **更新协议**：每解一题（无论对错）都加进经验日志，把规律提炼到模板表 / 谐音对照表里。

---

## 1. 主流程（v3）

```
input: 当前题截图

1. 视觉提取（无脑必做）
   - 题号、类型徽章、答案字数、上图 caption、下图与上图的差异

2. 谐音模板匹配（5 类）
   A 组合谐音 / B 同音异调 / C 场景成语 / D 双重谐音 / E 整字替换

3. 逐字对应推理（关键!）
   把答案的每个字都对应到视觉中的某个元素或动作
   能一一映射 → confidence 高 → 第一枪押
   有字找不到出处 → confidence 低 → TAB 拿提示

4. 第一枪策略
   - confidence > 0.85 且字数/类型对得上 → 直接押
   - confidence < 0.7 或类型不熟 → 先 TAB 拿第 1 字拼音锚点

5. 反馈解析（Wordle 三色）
   - 全绿 → 通关 → 按 Enter 进下一题
   - 有橙色 → 该字/拼音在答案里但位置错 → 重排
   - 全灰 → 完全错方向 → 不要近邻替换

6. 知识库记账
   answers.jsonl: {题号, 答案, 类型, 尝试次数, 错误候选, 是否用提示, 谐音解释}
```

## 5. UI / 工程避坑（实战累积）

| 行为 | 推荐方式 | 备注 |
|---|---|---|
| 截屏 | Steam 启动选项加 `--disable-gpu` | 不加抓到全黑（GPU 直渲染绕过截屏 API） |
| 切下一题 | **按 Enter** | 点「恭喜通关」按钮经常无效（焦点问题） |
| 输入答案 | click 输入框 → type 中文 → Enter | 走 clipboard fast path |
| 用提示 | TAB → click/Enter | 每按一次揭示下一字（先拼音再字符）；不算错误 |
| 退出提示 | 再按 TAB | 揭示的字会消失，必须自己重打 |
| 中文输入 | **直接写中文字符** | 别手算 \uXXXX —— 我们栽在这上面 3 次 |
| 焦点丢失 | 用户手动点游戏窗口 | Windows 焦点保护，open_application 不能强制夺焦 |

## 6. 反模式（要避免！）

1. **盲信视觉直觉** — 第一枪不要押"和场景看起来很像"但谐音说不通的（No.84 错了"汪洋大海"）
2. **答案近邻替换** — 全灰说明方向错，别去试同候选的近义词
3. **默认 4 字** — 题目字数从输入框空格数读
4. **忽略类型徽章** — 徽章决定答案池，绝不跨类型
5. **手算 unicode 转义** — 直接写中文字符，让 JSON 自己 encode
6. **静等自动跳关** — 实测自动跳不可靠，主动按 Enter

## 7. 经验日志（按解题时间倒序）

> 完整解题记录见 `answers.jsonl`，这里只记录有教学价值的。

### 最新批次（No.92-110）的关键洞见
- **首次见到橙色（No.92）**：「正方形」错答给出 方=橙色，立刻锁定方在 pos 1/3 → 第二枪「方程式」
- **反讽谐音首例（No.106）**：禁止步 → 进步（jìn 同音；视觉 negative，答案 positive）
- **全字逐位映射的高级题（No.93 求同存异、No.99 匪夷所思）** 都是一击命中 — 当每个字都能找到出处时 confidence 极高
- **类型库扩张** — 新增 评价/娱乐/玩具/国学/地质时期/动物/基础设施 等冷门类型

### 早期批次（No.82-91）的关键洞见
- **No.84 过目不忘**：教训案例 — 视觉直觉「汪洋大海」错了，真答案是同音异调 不汪→不忘
- **No.87 碌碌无为**：双重谐音首例 — 鹿鹿+无围
- **No.91 沈阳**：上图主体某字被下图动作的同音字替换（太→沈）

## 8. 整体战绩（截至 No.110）

```
通关率: 29/29 = 100%
一击命中(零错零提示): 16/29 = 55%
零错误尝试(可能用提示): 21/29 = 72%
其中 unicode 笔误造成的错答: 3 题（不算 AI 推理错）
扣除笔误后真实一击率: 19/29 = 66%
```

**结论**：AI 推理本身在 ≥66% 的题上一击命中，剩下的题用 1-2 次 TAB 提示也能稳定推出来。这套 prompt + 反馈循环已经够强。
